Chapter 22: Mongoose Introduction & Schema

What is Mongoose?

Mongoose is an Object Data Modeling (ODM) library for MongoDB and Node.js. It provides a structured way to interact with MongoDB by using schemas and models.

Why Use Mongoose?

  • Simplifies MongoDB operations
  • Provides schema-based structure
  • Built-in validation
  • Easy integration with Express

Installing Mongoose

npm install mongoose

Connecting Node.js with MongoDB

const mongoose = require("mongoose");

mongoose.connect("mongodb://localhost:27017/mernDB")
  .then(() => console.log("MongoDB Connected"))
  .catch(err => console.log(err));

What is a Schema?

A schema defines the structure of documents inside a MongoDB collection. It specifies field names, data types, and validation rules.

Creating a Schema

const mongoose = require("mongoose");

const studentSchema = new mongoose.Schema({
  name: String,
  age: Number,
  course: String
});

What is a Model?

A model is a JavaScript class created from a schema. It represents a collection in MongoDB and is used to perform CRUD operations.

Creating a Model

const Student = mongoose.model("Student", studentSchema);

Inserting Data Using Model

const student1 = new Student({
  name: "Amit",
  age: 21,
  course: "MERN"
});

student1.save()
  .then(() => console.log("Student saved"))
  .catch(err => console.log(err));

Schema with Validation

const userSchema = new mongoose.Schema({
  username: {
    type: String,
    required: true
  },
  email: {
    type: String,
    unique: true
  },
  age: {
    type: Number,
    min: 18
  }
});

Benefits of Mongoose

  • Cleaner and structured code
  • Data validation
  • Easy relationship handling

Summary

  • Mongoose connects Node.js with MongoDB
  • Schemas define data structure
  • Models perform database operations
