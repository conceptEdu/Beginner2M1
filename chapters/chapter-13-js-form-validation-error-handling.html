Chapter 13: JavaScript Form Validation & Error Handling

What is Form Validation?

Form validation is the process of checking user input before submitting data to the server. It helps ensure correct, complete, and secure data.

Why Form Validation is Important?

  • Prevents invalid data submission
  • Improves user experience
  • Reduces server load
  • Enhances application security

Client-Side Form Validation

Client-side validation is performed in the browser using JavaScript before the data is sent to the server.

Simple Validation Example

<form onsubmit="return validateForm()">
  <input type="text" id="username" placeholder="Username">
  <button type="submit">Submit</button>
</form>

<script>
function validateForm() {
  let user = document.getElementById("username").value;
  if (user === "") {
    alert("Username is required");
    return false;
  }
  return true;
}
</script>

Validating Multiple Fields

function validateForm() {
  let email = document.getElementById("email").value;
  let password = document.getElementById("password").value;

  if (email === "" || password === "") {
    alert("All fields are mandatory");
    return false;
  }
  return true;
}

Email Validation Using Regex

function validateEmail(email) {
  let pattern = /^[^ ]+@[^ ]+\.[a-z]{2,3}$/;
  return pattern.test(email);
}

Error Handling in JavaScript

Error handling is used to manage runtime errors and prevent application crashes.

try – catch – finally

Syntax

try {
  // code that may cause error
} catch (error) {
  // handle error
} finally {
  // always executes
}

Example

try {
  let x = y + 10;
} catch (err) {
  console.log("Error occurred:", err.message);
} finally {
  console.log("Execution completed");
}

Throwing Custom Errors

function checkAge(age) {
  if (age < 18) {
    throw "Age must be 18 or above";
  }
  return "Access granted";
}

try {
  console.log(checkAge(15));
} catch (e) {
  console.log(e);
}

Summary

  • Form validation checks user input
  • JavaScript performs client-side validation
  • Error handling prevents crashes
  • try–catch manages runtime errors
