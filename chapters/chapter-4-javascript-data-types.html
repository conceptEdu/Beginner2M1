Chapter 4: JavaScript Data Types

What are Data Types?

Data types specify the type of data that a variable can store. JavaScript is a dynamically typed language, which means the data type of a variable is decided at runtime.

Classification of Data Types

  • Primitive Data Types
  • Non-Primitive (Reference) Data Types

1. Primitive Data Types

Primitive data types store single values and are immutable.

a) Number

The Number data type is used to store numeric values (both integers and decimals). 

let a = 10;
let b = 3.5;
console.log(a, b);

b) String

The String data type is used to store text. Strings are written inside single or double quotes. 

let name = "JavaScript";
let msg = 'Hello World';
console.log(name, msg);

c) Boolean

The Boolean data type has only two values: true or false. 

let isLoggedIn = true;
let isAdmin = false;
console.log(isLoggedIn, isAdmin);

d) Undefined

A variable declared but not assigned any value has the undefined data type. 

let x;
console.log(x);   // Output: undefined

e) Null

The null data type represents an intentional absence of value. 

let data = null;
console.log(data);

2. Non-Primitive (Reference) Data Types

Non-primitive data types can store multiple values and are mutable.

a) Object

Objects store data in key-value pairs. 

let student = {
  name: "Amit",
  age: 20,
  course: "MERN"
};
console.log(student);

b) Array

Arrays store multiple values in a single variable. 

let subjects = ["HTML", "CSS", "JavaScript"];
console.log(subjects);

c) Function

Functions are reusable blocks of code that perform a specific task. 

function greet() {
  console.log("Welcome to MERN Stack");
}
greet();

Difference Between Primitive and Non-Primitive

Primitive Non-Primitive
Stores single value Stores multiple values
Immutable Mutable
Example: Number, String Example: Object, Array

Summary

  • JavaScript supports primitive and non-primitive data types
  • Primitive data types store simple values
  • Non-primitive data types store complex data
