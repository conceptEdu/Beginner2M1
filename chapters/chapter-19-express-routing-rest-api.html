Chapter 19: Express Routing & REST API Basics

What is Routing?

Routing refers to defining how an application responds to client requests for specific URLs (routes) and HTTP methods. Express provides a simple and powerful routing mechanism.

Basic Route Example

const express = require("express");
const app = express();

app.get("/", (req, res) => {
  res.send("Home Page");
});

app.listen(3000);

HTTP Methods in Express

  • GET – Retrieve data
  • POST – Send data to server
  • PUT – Update existing data
  • DELETE – Remove data

GET Method

app.get("/users", (req, res) => {
  res.send("List of users");
});

POST Method

app.post("/users", (req, res) => {
  res.send("User created");
});

PUT Method

app.put("/users/:id", (req, res) => {
  res.send("User updated");
});

DELETE Method

app.delete("/users/:id", (req, res) => {
  res.send("User deleted");
});

Route Parameters

Route parameters are used to capture values from the URL. 

app.get("/product/:id", (req, res) => {
  res.send(req.params.id);
});

What is REST API?

REST (Representational State Transfer) is an architectural style used to build web services. A REST API allows communication between client and server using HTTP methods.

REST API Characteristics

  • Client–Server architecture
  • Stateless communication
  • Uses HTTP methods
  • Returns JSON data

Simple REST API Example

app.get("/api/students", (req, res) => {
  res.json([
    { id: 1, name: "Amit" },
    { id: 2, name: "Neha" }
  ]);
});

Difference Between Traditional Routing and REST API

Traditional Routing REST API
Returns HTML Returns JSON
Used for webpages Used for client-server communication

Summary

  • Routing defines application endpoints
  • REST API uses HTTP methods
  • Express simplifies API development
