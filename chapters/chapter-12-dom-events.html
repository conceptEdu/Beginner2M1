Chapter 12: DOM Events & Event Handling

What are Events?

Events are actions or occurrences that happen in the browser when a user interacts with a webpage. JavaScript can respond to these events using event handlers.

Common DOM Events

  • click
  • dblclick
  • mouseover
  • mouseout
  • keydown
  • keyup
  • change
  • submit
  • load

Event Handling Methods

1. Inline Event Handling

Event is handled directly inside the HTML element. 

<button onclick="showMessage()">Click Me</button>

<script>
function showMessage() {
  alert("Button clicked!");
}
</script>

2. Event Handler Property

Events are handled by assigning a function to an element property. 

<button id="btn">Click</button>

<script>
document.getElementById("btn").onclick = function() {
  alert("Button clicked");
};
</script>

3. addEventListener() Method

This is the most recommended and modern way to handle events. It allows multiple events on a single element.

Syntax:

element.addEventListener("event", function);

Example:

<button id="myBtn">Click Me</button>

<script>
document.getElementById("myBtn")
  .addEventListener("click", function() {
    alert("Event triggered!");
  });
</script>

Event Object

When an event occurs, JavaScript automatically creates an event object that contains information about the event. 

document.addEventListener("click", function(event) {
  console.log(event.type);
  console.log(event.target);
});

Form Events

submit Event

<form id="myForm">
  <input type="text">
  <button type="submit">Submit</button>
</form>

<script>
document.getElementById("myForm")
  .addEventListener("submit", function(e) {
    e.preventDefault();
    alert("Form submitted");
  });
</script>

Difference Between onclick and addEventListener

onclick addEventListener
Only one event at a time Multiple events allowed
Old method Modern method

Summary

  • Events represent user interactions
  • JavaScript can handle events dynamically
  • addEventListener is the best practice
